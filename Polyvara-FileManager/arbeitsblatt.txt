🧠 Features
Aktion	Verhalten
open	Öffnet mit xdg-open
play	Erkennt .mp3, .mp4, .mkv, usw. und nutzt mpv/vlc
cat	Nur Textdateien werden angezeigt, Bilddateien geben Warnung aus
nano/vim…	Öffnet Datei in gewähltem Editor
delete	Fragt nach Bestätigung, dann rm
cp, mv…	Standard-Linux-Kommandos (wirken direkt auf Datei)
kein Befehl	Gibt Info, was verfügbar ist

source /pfad/zum/suche.sh
alias suche='bash /pfad/zum/suche.sh'


🧰 Voraussetzungen (alle sind Standard)

    xdg-open, mpv, vlc

    nano, vim, kate, etc.

    sudo für automatische VLC-Installation

    Deine Shell muss realpath und xargs können (Standard)

🧪 Erweiterbar?

Absolut. Du kannst danach alles durchreichen:
z. B. suche Download/- abc.txt chmod +x geht auch.
Oder suche Download/- server.log tail usw.


bashbeispiel :

#!/bin/bash
# such.sh: Projektweite Textsuche

function suche() {
    if [[ -z "$1" ]]; then
        echo "⚠️  Gib 'Ordner/- Suchbegriff' oder nur 'Suchbegriff' ein!"
        return 1
    fi

    # Default-Werte
    local suchordner="."
    local suchbegriff=""

    # Prüfen ob Format: pfad/- suchbegriff
    if [[ "$1" == *"/-" ]]; then
        suchordner="${1%/-}"     # Alles vor /-
        suchbegriff="$2"
    else
        suchbegriff="$1"
    fi

    if [[ -z "$suchbegriff" ]]; then
        echo "⚠️  Kein Suchbegriff erkannt!"
        return 1
    fi

    echo "🔍 Suche in '$suchordner' nach: '$suchbegriff'"
    echo "-----------------------------------------------"

    find "$suchordner" -type f ! -path '*/\.*' \
        -exec grep --color=always -niH "$suchbegriff" {} + 2>/dev/null

    echo "-----------------------------------------------"
    echo "🔚 Suche abgeschlossen."
}
